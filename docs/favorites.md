# Избранное: локальное хранение по языкам

**Статус:** спецификация. **Владельцы:** otus_dz/recipe_list.
**См. также:** [todo/15-favorites.md](../todo/15-favorites.md).

## Цель

Дать пользователю возможность отметить рецепт как избранный с карточки
или со страницы деталей и просматривать отмеченные рецепты на вкладке
«Избранное» в нижней навигации. Избранное переживает перезагрузку ленты
(случайно посеянная лента стирается кнопкой reload — избранное
стираться НЕ должно).

## UX

### Поставить / снять отметку

* На картинке карточки рецепта, в правом верхнем углу, рисуется иконка
  сердца в той же круглой 40×40-плашке и позиции, что и существующий
  YouTube-бейдж `_YoutubeBadge` (полупрозрачный чёрный фон, белый глиф).
* Контурное сердце (`Icons.favorite_border`), пока рецепт не в
  избранном.
* Тап переключает состояние. В избранном — заполненное зелёное сердце
  (`Icons.favorite`, `AppColors.primary`).
* Тот же бейдж появляется на hero-изображении страницы деталей
  (правый верхний угол) с идентичным поведением.

### Экран «Избранное»

* Вкладка нижней навигации «Избранное» (уже есть как
  `AppNavTab.favorites`, `Icons.favorite_border`) ведёт на
  `FavoritesPage`.
* Отрисовывает сетку рецептов в стиле `RecipeListPage` в обратном
  хронологическом порядке (последние добавленные — сверху).
* Пустое состояние: локализованная подсказка «Пока нет избранного».
* AppBar использует ту же оболочку `SearchAppBar`, что и вкладка
  «Рецепты», но кнопки **«язык»** и **«reload»** отрисованы
  приглушёнными (~38% непрозрачности) и неинтерактивными
  (`onPressed: null` / `IgnorePointer`). Они видимы для согласованности
  лэйаута, но инертны: язык неактуален (избранное привязано к языку,
  на котором сохранено), перезагружать нечего.
* Поле поиска **активно** и фильтрует список избранного локально
  (подстрочное совпадение на стороне клиента по `recipe.name` среди
  строк, возвращённых `FavoritesStore.list(appLang.value)`). Сетевой
  search API НЕ вызывается, поиск НЕ выходит за пределы избранного.
* Тап по карточке открывает существующий `RecipeDetailsPage`. Бейдж
  сердца там остаётся интерактивным и синхронизирует состояние через
  тот же стор.

## Модель хранения

Избранное хранится только локально, в той же sqflite-БД, что и кэш
рецептов. На сервере на этом этапе понятие «избранное пользователя»
отсутствует.

### Привязка к языку

Избранное хранится **по языкам** — намеренное упрощение: каждая пара
`(recipe_id, lang)` это отдельная строка. Обоснование:

* Пользователь обычно остаётся в родном языке. Хранить все локали для
  каждого избранного — это десятикратные затраты на БД и переводы
  ради функции, которой пользуются изредка.
* Избранное фиксирует рецепт **в том виде, в каком его увидел
  пользователь** в момент клика. Перевод при смене языка — явный
  выбор: переключился на TR, нажал сердце ещё раз — «сохранил
  по-турецки».
* Экран «Избранное» отрисовывает только строки, где `lang` совпадает
  с текущим `appLang`. Переключение на язык, в котором избранного
  нет, показывает пустое состояние.

### Схема

Таблица `favorites` в существующей `recipe_db` (поднять
`kRecipeDbSchemaVersion` с 5 до 6):

```sql
CREATE TABLE favorites (
  recipe_id INTEGER NOT NULL,
  lang TEXT NOT NULL,
  saved_at INTEGER NOT NULL,  -- миллисекунды от эпохи
  PRIMARY KEY (recipe_id, lang)
);
CREATE INDEX favorites_by_lang_savedAt
  ON favorites (lang, saved_at DESC);
```

Тело рецепта уже кэшируется в `recipe_bodies` (`recipe_id`, `lang`);
таблица favorites хранит только факт принадлежности. На экране
«Избранное» делаем `JOIN recipe_bodies USING (recipe_id, lang)` для
отрисовки. Если строка тела вытеснилась кэшем — фолбек на сетевой
`lookup` (избранное достаточно редкое событие, чтобы это было
приемлемо).

### Миграция

`onUpgrade(5 → 6)`: `CREATE TABLE favorites` + индекс. Бэкфилла нет.

## Управление состоянием

Один синглтон `FavoritesStore` (по образцу `ChangeNotifier`, как и
существующие нотифаеры `appLang` / `reloadingFeed`):

* `Future<void> add(int recipeId, AppLang lang)`
* `Future<void> remove(int recipeId, AppLang lang)`
* `Future<bool> isFavorite(int recipeId, AppLang lang)`
* `ValueListenable<Set<int>> idsForLang(AppLang lang)` — UI
  переподписывается при смене языка.
* `Future<List<Recipe>> list(AppLang lang)` — джойнит
  `recipe_bodies` для экрана «Избранное».

Бейдж сердца подписан на `idsForLang(appLang.value)` и перерисовывается
при insert/remove.

## Что НЕ входит в scope

* Синхронизация между устройствами. Избранное на сервере не живёт.
* Миграция избранного между языками (нет режима «показать мои
  IT-избранные, пока UI в TR» — by design).
* Папки / теги / заметки на избранном.
* Массовое «добавить в избранное всю категорию».
* iOS-стиль swipe-to-delete на экране избранного (для v1 достаточно
  long-press меню).
* Серверная аналитика событий избранного.

## Критерии приёмки

* Бейдж сердца отрисован на каждой карточке рецепта и на странице
  деталей в той же позиции и визуальном весе, что и YouTube-бейдж.
* Тап переключает контурное ↔ заполненное-зелёное за один кадр;
  состояние сохраняется при перезапуске приложения.
* Таблица `favorites` создаётся при первом запуске после апгрейда;
  существующие кэши НЕ очищаются.
* Кнопка reload на вкладке «Рецепты» НЕ трогает таблицу `favorites`
  (на вытеснение при пересеве идут только `recipes` / `recipe_bodies`).
* Экран «Избранное» отрисовывает только строки, где
  `lang == appLang.value`. Смена языка обновляет экран в реальном
  времени.
* `flutter analyze` чистый. Новые unit-тесты для `FavoritesStore`
  плюс widget-тест переключения сердца.
